# HashiCorp 的设计系统灵感

## 1.视觉主题和氛围

HashiCorp 的网站是有形的企业基础设施 - 一个设计系统，必须传达云基础设施管理的复杂性，同时保持平易近人。视觉语言分为两种模式：用于信息部分的干净白光模式和用于英雄区域和产品展示的戏剧性黑暗模式（“#15181e”、“#0d0e12”），创建了昼夜二元性，反映了“在光明中构建，在黑暗中部署”开发人员工作流程。

版式以自定义品牌字体（HashiCorp Sans，加载为“__hashicorpSans_96f0ca”）为基础，字面上具有很大的分量。标题使用 600-700 的粗细和严格的行高 (1.17-1.19)，创建密集、权威的文本块来传达企业信心。启用 OpenType“kern”的 82px 粗细 600 的英雄标题不是装饰性的 - 它是基础设施级的排版。

HashiCorp 的独特之处在于其多产品色彩系统。产品组合中的每个产品都有自己的品牌颜色 - Terraform 紫色 (`#7b42bc`)、Vault 黄色 (`#ffcf25`)、Waypoint 青色 (`#14c6cb`)、Vagrant 蓝色 (`#1868f2`) - 这些颜色通过 CSS 自定义属性系统 (`--mds-color-*`) 作为重音标记出现。这在设计系统中创建了一个设计系统：母品牌是黑白的，带有蓝色的点缀，而每个子产品都注入了自己的色彩特征。

组件系统使用“mds”（Markdown 设计系统）前缀，表示一种系统的、标记驱动的方法，其中颜色、间距和状态都通过 CSS 变量进行管理。阴影非常微妙 - 使用“rgba(97, 104, 117, 0.05)”的双层微阴影几乎不可见，但提供了足够的深度来将交互表面与背景分开。

**主要特征：**
- 双模式：干净的白色部分 + 戏剧性的黑暗 (`#15181e`) 英雄/产品区域
- 自定义 HashiCorp Sans 字体，具有 600–700 粗细和“kern”功能
- 通过 `--mds-color-*` CSS 自定义属性的多产品颜色系统
- 产品品牌颜色：Terraform 紫色、Vault 黄色、Waypoint 青色、Vagrant 蓝色
- 大写字母间距标题（13 像素，粗细 600，1.3 像素字母间距）
- 微阴影：不透明度为 0.05 的双层 — 通过耳语而不是喊叫来实现深度
- 具有语义变量名称的令牌驱动的“mds”组件系统
- 严格的边框半径：2px–8px，没有药丸状或圆形
- 辅助文本的系统 UI 后备堆栈

### 原则
- **品牌/系统分割**：HashiCorp Sans 用于标题和品牌关键文本；用于正文、导航和功能文本的 system-ui。品牌字体承载重量，system-ui 承载文字。
- **Kern 始终开启**：所有 HashiCorp Sans 文本都启用 OpenType“kern”——字母调整是不可协商的。
- **紧凑的标题**：每个标题都使用 1.17-1.21 行高，创建密集、堆叠的文本块，给人一种基础设施的感觉——坚固、承重。
- **宽松的正文**：正文使用 1.50–1.69 的行高（特别宽大），在密集的标题下营造舒适的阅读节奏。
- **大写标签作为寻路**：13px 大写字母和 1.3px 字母间距用作系统类别/部分标记 - 始终为 HashiCorp Sans 粗细 600。

## 5.布局原则

### 间距系统
- 基本单位：8px
- 比例：2 像素、3 像素、4 像素、6 像素、7 像素、8 像素、9 像素、11 像素、12 像素、16 像素、20 像素、24 像素、32 像素、40 像素、48 像素

### 网格和容器
- 最大内容宽度：~1150px（xl 断点）
- 包含内容的全角黑暗英雄部分
- 卡片网格：2–3 列布局
- 桌面规模的宽大水平填充

|名称 |宽度|关键变化|

|移动小| <375 像素 |紧单柱|
|手机 | 375–480 像素 |标准移动|
|小型平板电脑| 480–600 像素 |小幅调整|

### 空白哲学
- **企业呼吸空间**：各部分之间宽敞的垂直间距（48px–80px+）传达了稳定性和严肃性。
- **密集的标题，宽敞的正文**：紧密的行高标题位于宽松的正文之上，在每个部分中创造视觉上的“顶部重量”。
- **深色如画布**：深色英雄部分使用额外的垂直填充，让 3D 插图和渐变呼吸。

### 边界半径比例
- 最小（2px）：链接、小内联元素
- 微妙（3px）：复选框，小输入
- 标准 (4px)：辅助按钮
- 舒适 (5px)：主要按钮、徽章、输入
- 卡片 (8px)：卡片、容器、图像

## 6. 深度和高度

|水平|治疗 |使用 |
|--------|------------|-----|
|公寓（0 级）|没有影子|默认表面、文本块|
|耳语（1 级）| `rgba(97, 104, 117, 0.05) 0px 1px 1px, rgba(97, 104, 117, 0.05) 0px 2px 2px` |卡片、按钮、交互界面|
|焦点（2 级）| `3px 实心 var(--mds-color-focus-action-external)` 轮廓 |对焦环 — 颜色与环境相匹配 |

**影子哲学**：HashiCorp 可以说是现代网页设计中最微妙的影子系统。不透明度为 5% 的双层阴影几乎是看不见的——它们的存在不是为了创造视觉深度，而是为了传达交互性。如果你能看到影子，那就太强了。这种克制传达了企业稳定的价值观——没有什么是浮动的，没有什么是不确定的。

## 7. 该做和不该做的事情

### 做
- 使用 HashiCorp Sans 作为标题和品牌文本，使用 system-ui 作为正文和 UI 文本
- 在所有 HashiCorp Sans 文本上启用“kern”
- 仅对各自的产品使用产品品牌颜色（Terraform = 紫色，Vault = 黄色等）
- 以 13 像素粗细 600 应用大写标签，部分标记的字母间距为 1.3 像素
- 将阴影保持在“耳语”级别（双层不透明度 0.05）
- 使用“--mds-color-*”令牌系统来实现一致的颜色应用
- 保持头紧/身体放松的节奏（1.17–1.21 vs 1.50–1.69 线高）
- 使用“3pxsolid”焦点轮廓以实现可访问性

### 不要
- 不要在产品上下文之外使用产品品牌颜色（Vault 内容上不得使用 Terraform 紫色）
- 不要将阴影不透明度增加到 0.1 以上 - 耳语级别是故意的
- 不要使用药丸形状的按钮（>8px半径）——尖锐、最小的半径是结构性的
- 不要跳过标题上的“kern”功能——字体需要它
- 不要将 HashiCorp Sans 用于小正文文本 — 它是为 17px+ 标题使用而设计的
- 不要在同一组件中混合产品颜色 - 每个产品都有一种颜色
- 不要将纯黑色 (`#000000`) 用于深色背景 — 使用 `#15181e` 或 `#0d0e12`
- 不要忘记不对称的按钮填充 - 9px 9px 9px 15px 是故意的

## 8. 响应行为

### 断点
|名称 |宽度|关键变化|
|------|--------|-------------|
|手机 | <768 像素 |单栏、汉堡式导航、堆叠 CTA |
|平板电脑| 768–992 像素 | 2 列网格，导航开始扩展 |
|桌面| 992–1150 像素 |完整布局，大型菜单导航 |
|大| >1150 像素 |最大宽度居中、宽边距 |

### 崩溃策略
- 英雄：82px → 52px → 42px 标题大小
- 导航：超级菜单→汉堡包
- 产品卡：3 列 → 2 列 → 堆叠
- 深色部分保持全宽但压缩填充
- 按钮：内联 → 全角堆叠在移动设备上

### 迭代指南
1. 始终从模式决定开始：浅色（白色）用于信息，深色（#15181e）用于英雄/产品
2. HashiCorp Sans 仅用于标题（17px+），system-ui 用于其他所有内容
3. 阴影处于耳语级别（0.05 不透明度）——如果可见，则减少
4. 产品颜色是神圣的——每件产品都只有一种颜色
5. 对焦环始终为 3px 实心，颜色与产品背景相匹配
6. 大写标签是系统的寻路模式——13px、600、1.3px 跟踪
